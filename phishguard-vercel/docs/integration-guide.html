Integration Details
01
🔌 Chrome Extension
Injects a scan button + threat banner directly into Gmail's UI. Works instantly in the browser — no server config needed.
1
Open Chrome → chrome://extensions
2
Enable Developer mode (top right toggle)
3
Click Load unpacked → select the extension/ folder
4
Edit content.js line 1: set API_BASE to your server URL
5
Open Gmail — PhishGuard scans every email you open automatically

02
📬 SMTP Milter
Server-level filter. Scans every email before delivery and can reject, quarantine, or label threats automatically.
1
Install: pip install pymilter requests
2
Configure Postfix main.cf: smtpd_milters = inet:127.0.0.1:8894
3
Set thresholds in milter: REJECT_ON = ["CRITICAL"]
4
Run: python phishguard_milter.py
5
Test: sendmail -v [email protected] < test.eml
RiskAction
CRITICALReject (550 error)
HIGHMove to Spam folder
MEDIUMPrepend [SUSPECTED PHISHING]
SAFEDeliver with X-PhishGuard header
03
🔗 Gmail Webhook (Pub/Sub)
Uses Gmail API push notifications to auto-scan every new email and apply labels/filters programmatically.
1
Enable Gmail API + Pub/Sub in Google Cloud Console
2
Create Pub/Sub topic: phishguard-gmail-notifications
3
Create push subscription → https://yourserver.com/webhook/gmail
4
Run setup_gmail_watch.py to register inbox watching
5
Deploy phishguard_webhook.py (Flask server on port 5050)
6
Labels are auto-created in Gmail: PhishGuard/Phishing, PhishGuard/Suspected
📄 phishguard_webhook.py
📄 setup_gmail_watch.py
📄 credentials.json
04
📱 Gmail Add-in
Official Google Workspace add-in that shows a PhishGuard panel in the reading pane. Deployable org-wide via admin console.
1
Go to script.google.com → New project
2
Replace Code.gs with the provided Apps Script code
3
Replace appsscript.json with the provided manifest
4
Set PHISHGUARD_API_URL to your deployed API endpoint
5
Deploy → New deployment → Gmail Add-on → Publish
6
For org-wide: Admin console → Apps → Marketplace apps → Domain install

📄 appsscript.json
Prerequisites
🐍 Backend Server
  • Python 3.10+ with FastAPI running
  • Accessible via public URL (or localhost for extension)
  • PhishGuard backend from phishguard_backend.py
  • Optional: Docker / Nginx for production
☁️ Google Cloud (Webhook)
  • GCP project with billing enabled
  • Gmail API + Cloud Pub/Sub APIs enabled
  • OAuth2 credentials (credentials.json)
  • Pub/Sub topic + push subscription
🖥 Chrome Extension
  • Google Chrome or Chromium browser
  • Developer mode enabled in extensions
  • PhishGuard API running (local or deployed)
  • No signup or store submission needed
📬 SMTP Milter
  • Access to your mail server (Postfix/MTA)
  • pymilter Python package
  • For Google Workspace: use routing rules instead
  • Linux server recommended

pip install fastapi uvicorn pymilter requests flask google-auth google-api-python-client google-cloud-pubsub
python phishguard_backend.py
      → API running at http://localhost:8000

# 2. Load Chrome extension
      → chrome://extensions → Load unpacked → select extension/ folder

# 3. Start the Pub/Sub webhook listener

python phishguard_webhook.py  # keep running
